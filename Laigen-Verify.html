Instructions for verifying Laigen2.zip


Download both Laigen2.zip and Laigen2.zip.asc and place them in the same folder.

OpenPGP-compliant software is needed to verify authenticity. GnuPG is what will be used in the instructions below. If you're using another kind of OpenPGP-compliant software, you should look for instructions written specifically for your choice of software.

If you're using GnuPG, the instructions below apply to all platforms when using a terminal, even on mobile.

Import signing key

gpg --keyserver keys.openpgp.org --recv-key 9578F98FA389921993A04A742D41270933F04B9B

Verify that the key was imported correctly

gpg --list-key --with-fingerprint 2D41270933F04B9B

The output should look like this:

pub   rsa4096 2026-03-12 [SC]
      9578 F98F A389 9219 93A0  4A74 2D41 2709 33F0 4B9B
uid           [ unknown] Brandon Cullen <bjacullen01@proton.me>
sub   rsa4096 2026-03-12 [E]

MAKE SURE that the signing key's fingerprint, shown below:

9578 F98F A389 9219 93A0  4A74 2D41 2709 33F0 4B9B

matches the fingerprint in your output.


Verify authenticity of Laigen2.zip

Make sure you're in the same directory as Laigen2.zip and Laigen2.zip.asc!

gpg --verify Laigen2.zip.asc Laigen2.zip

You should see this output:

gpg: Signature made Sat 09 May 2026 12:01:51 AM EDT
gpg:                using RSA key 9578F98FA389921993A04A742D41270933F04B9B
gpg:                issuer "bjacullen01@proton.me"
gpg: Good signature from "Brandon Cullen <bjacullen01@proton.me>" [unknown]

Note

GPG will most likely show you a warning, right under the output shown above, that the key's User ID is not certified with a trusted signature.

This is expected, and is not a problem!

As long as the output shows:

Good signature from "Brandon Cullen <bjacullen01@proton.me>" [unknown]

and the key's fingerprint is:

9578 F98F A389 9219 93A0 4A74 2D41 2709 33F0 4B9B

then you've verified that Laigen2.zip is intact and has not been tampered with!
