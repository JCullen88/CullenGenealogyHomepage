Instructions for verifying Laigen2.zip on Linux


Download both Laigen2.zip and Laigen2.zip.asc and place them in the same folder.

Make sure you have OpenPGP-compliant software installed. Most Linux-based distros come with GnuPG installed already, so that is what will be used in the examples here.

Import signing key

gpg --keyserver hkp://keys.openpgp.org:80 --recv-key EFBA7D9EC0EE464C489DB45F8585F6FECA38F889

Verify that the key was imported correctly

gpg --list-key --with-fingerprint 8585F6FECA38F889

The output should look like this:

pub   rsa4096 2026-04-13 [SC] [expires: 2028-04-12]
      EFBA 7D9E C0EE 464C 489D  B45F 8585 F6FE CA38 F889
uid           [ unknown] Brandon Cullen (Signing Key) <bjacullen01@proton.me>

MAKE SURE that the signing key's fingerprint, shown below:

EFBA 7D9E C0EE 464C 489D  B45F 8585 F6FE CA38 F889

matches the fingerprint in your output.


Verify authenticity of Laigen2.zip

Make sure you're in the same directory as Laigen2.zip and Laigen2.zip.asc!

gpg --verify Laigen2.zip.asc Laigen2.zip

You should see this output:

gpg: Signature made Mon 13 Apr 2026 02:50:41 PM EDT
gpg:                using RSA key EFBA7D9EC0EE464C489DB45F8585F6FECA38F889
gpg:                issuer "bjacullen01@proton.me"
gpg: Good signature from "Brandon Cullen (Signing Key) <bjacullen01@proton.me>" [unknown]

Note

GPG will most likely show you a warning, right under the output shown above, that the key's user ID is not certified with a trusted signature.

This is expected, and is not a problem!

As long as the output shows:

Good signature from "Brandon Cullen (Signing Key) <bjacullen01@proton.me>" [unknown]

and the key's fingerprint is:

EFBA 7D9E C0EE 464C 489D B45F 8585 F6FE CA38 F889

then you've verified that Laigen2.zip is all good!
